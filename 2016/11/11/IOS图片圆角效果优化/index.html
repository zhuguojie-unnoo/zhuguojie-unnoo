IOS图片圆角效果优化

显示为圆角的图片现在很常见，之前使用的时候一直没有考虑过效率问题，因为系统提供了属性，直接设置就完事了。（ps：在优化前我一直是使用这种方法来实现的，简单快捷）导致性能不好的原因主要是引起了 offscreen rendering.

造成 offscreen rendering的原因有：

* Any layer with a mask (layer.mask)
* Any layer with layer.masksToBounds being true
* Any layer with layer.allowsGroupOpacity set to YES and layer.opacity is less than 1.0
* Any layer with a drop shadow (layer.shadow*).
* Any layer with layer.shouldRasterize being true
* Any layer with layer.cornerRadius, layer.edgeAntialiasingMask, layer.allowsEdgeAntialiasing
优化的过程中尽量不要使用上面的属性或者方法 ### 1、使用系统的属性 设置圆角的半径，设置超出部分剪裁。 
imageView.layer.cornerRadius = 20;
imageView.masksToBounds = YES;
当然也有说设置 shouldRasterize 为YES,但是如果是在一个界面有“大量”的圆角图片的 tableView 或者 conllectionView 中时，是没有效果的。 ### 2、使用贝塞尔曲线画出 imageView 在分类中写一个方法，使用的时候直接调用该方法。 self.clipsToBounds = YES; UIBezierPath *path = [UIBezierPath bezierPathWithRoundedRect:CGRectMake(0, 0, radius * 2, radius * 2) cornerRadius:radius]; CAShapeLayer *maskLayer = [CAShapeLayer layer]; maskLayer.path = path.CGPath; self.layer.mask = maskLayer; ### 3、处理成圆角图片 在图片显示到界面上时，先把图片处理成员圆角图片。大方向有两种方案：服务器处理和客户端处理。服务器返回圆角图片在此我就不多说了，使用的是不会有性能问题的。我们来说说客户端将图片处理成圆角的方案。 #### a.从网络上拿到图片时将图片处理成圆角存放在本地 #### b.从网络上拿到图片存放在本地是原图，加载到内存或缓存时处理成圆角 QQ 采用就是从服务器获取存在本地是原图，显示的时候是圆角图片的方案的。 优点：界面不卡顿、代码量一般 缺点：此方法比较消耗 CPU 资源 ，代码需要时刻维护（如结合第三框架来修改，需要随时维护处理图片的代码） Tip: 以上两种处理图片的方法可以结合 SDWebImage 的第三方框架来实现，在对应的位置去处理图片就行。 
UIGraphicsBeginImageContextWithOptions(avatarImageView.bounds.size, NO, [UIScreen mainScreen].scale);
[[UIBezierPath bezierPathWithRoundedRect:avatarImageView.bounds
                              cornerRadius:50] addClip];
[image drawInRect:avatarImageView.bounds];
avatarImageView.image = UIGraphicsGetImageFromCurrentImageContext();
UIGraphicsEndImageContext();

4、在显示图片的控件上加上一个圆角遮罩

优点：性能好，不卡顿，代码量比较少

Tip: 需要注意圆角遮罩的颜色，必须与背景色相同，比如和 cell 的背景色相同。点击状态和非点击状态需要变换颜色与其保持一致。

总结

优化是权衡各种方案的利弊，然后结合自己的实际情况，选择对于自己项目来说最优的方案。

比如：界面上圆角图片比较少的情况（只有几个）那么直接使用第一种或者第二种都行，这几张图片的消耗不足于引起界面的卡顿。
